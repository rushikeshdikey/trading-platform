One-click — pull today's trades from Zerodha

Fetches every execution from your Zerodha account for today via Kite's API and feeds it through the same dedup + FIFO reconstruction pipeline as the xlsx upload. Safe to run multiple times — duplicates are skipped. Requires an active Kite session (tokens expire around 06:00 IST each day).

Option 1 — upload an xlsx

Any Excel file following the same DTrades layout.

Option 2 — Zerodha tradebook (CSV or XLSX)

Export from Zerodha Console → Reports → Tradebook. CSV or XLSX both work. Imports are append-only and deduplicated on Zerodha's Trade ID, so re-uploading an overlapping tradebook is safe. New executions for a symbol with an already-open trade extend that trade (pyramid / exit) instead of creating a duplicate.

Stop-loss and setup aren't in the broker export — fill those in per trade afterwards (the Edit button on the trade page).
